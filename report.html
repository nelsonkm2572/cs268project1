  • CLIENT CRITIQUE

1. My client was my girlfriend, who is a recent Biology graduate.
2. The header contains my client's name and the current page. On the phone layout, it also contains a menu icon.
3. There are five pages, not including the report page. They are the Home, AIS Work, Prairie Burn, Saving Bees, and Resume pages.
4. There are navigation links for every page in the menu bar.
5. For large screens, the menu bar is always visible and is aligned in 1 row. The content area is on the left half of the screen. For small screens, the menu bar is hidden unless the menu icon is pressed, where it is aligned in 1 column. The content area covers the width of the screen.
6. The style is the same for every page, except for different background images. The Resume and Report pages both have seperate style items, which are located in stylesresume.css and stylesreport.css.
7. The menu icon is an image with the alt set to "Menu Button".
8. The fav icon is a bee for every page.
9. All images are in an image folder, all css is in a styles folder, and all javascript is in a js folder.
10. I didn't use any libraries or frameworks to build the site.
11. All pages and css validated using the HTML and CSS validators.
12. The site is stored on a git repository that was shared to my instructor.
13. All files and folder are undercase.
14. The site is served on port 8443.
15. The site can only be accessed through HTTPS.
16. The site is aligned to display content on half the screen in a semi-transparent container so the background images can be seen. This is meant to tie the image with the content of the page.
17. The site uses Arial for the header and headings and uses Lucida Grande for content.
18. The home page has the option to send an email, which is done with a form.
19. The site has a link to Bee City USA, Freepik, and www.flaticon.com.
20. I used consistent formatting for all HTML, CSS, and JavaScript. I also used meaningful names for classes and variable names.
21. I use hover effects for menu items and links, as well as hiding and showing the menu for smaller screens.
